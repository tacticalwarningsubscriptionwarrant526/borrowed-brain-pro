**一个 Claude Skill，把任何公众人物的思考方式蒸馏成一套可复用的框架——然后把它当作一个额外的视角，套用到你正在面对的真实决策上。**

不是聊天机器人角色扮演，不是语录生成器。而是一份基于真实公开材料——访谈、决策记录、失败、批评——构建出的结构化「思维档案」，你可以拿它去照一照自己的问题：*这套框架会让我看到什么我原本忽略的东西？*

---

## 🚀 刚装好这个 skill？从这里开始

安装完之后（见下面的[安装](#安装)），开一个新对话，直接打以下任意一句即可——不需要特殊语法，不需要斜杠命令：

```
帮我建一份 Naval Ravikant 的思维档案
```
```
用 Reed Hastings 的档案，看看我这个招聘决定漏掉了什么——[描述你的情况]
```
```
borrowed-brain 是做什么的？
```

最后这句也完全没问题——如果你不确定该问什么，直接问这个 skill 本身就行，它会解释两种模式并直接给你一句能用的起手式，而不是让你自己猜。

---

## 为什么做这个

大多数"像 X 一样思考"的提示词产出的都是千篇一律的励志海报式空话——每份档案读起来都一样，因为它们只用了这个人最光鲜的公开形象来构建。

`borrowed-brain` 刻意去挖更混乱的材料——已记录的失败、批评、本人说法和外界说法对不上的地方——因为真正的决策逻辑正是在这些地方才会显现。它还强制要求档案里的每一条"原则"都要能追溯到一个具体的、有来源的事实，让档案保持扎实，而不是沦为吹捧。

## 它做什么

**Distill（蒸馏）模式** —— 给它一个名字，它会研究这个人并生成 `profiles/<name>.md`：核心立场、反复出现的原则（附带每条原则在什么条件下会失效）、默认推理顺序、取舍倾向、一个已记录的失败案例，以及一份说明材料扎实程度的可信度说明。

**Apply（应用）模式** —— 把一份已有档案套到你正面对的真实问题上，它会用这个视角来推理你的处境——并明确标注这只是"一个有局限的视角"，不是判决。

## 示例

以下是从 [`profiles/reed-hastings.md`](profiles/reed-hastings.md) 里摘出的真实片段，由实际运行 Distill 模式生成——不是虚构的样例：

> **原则**：主动征求异议，而不是把沉默当成同意（"farming for dissent"）。
> **体现之处**：直接源自 Qwikster 的失败案例（2011 年）——Hastings 后来发现多位副总裁私下都对拆分方案有疑虑却没说出来，其中一位告诉他，他一旦确信某件事就会"太intense"，让人觉得反对也没用。此后他把"隐瞒异议"视为员工的失职，而不只是一种需要绕开的性格特质。
> **可能失效之处**：这条原则是事后补救，不是事前预防——它是在一次公开的、导致 80 万订阅用户流失、股价暴跌 77% 的失败之后才被建立起来的，而不是之前。这说明这套机制可能无法在事发当下实时捕捉到"确信"带来的盲点，只能等代价已经付出之后才在事后起作用。

以下是 Apply 模式针对一个真实问题的应用，同时载入了另一份档案（[`profiles/travis-kalanick.md`](profiles/travis-kalanick.md)）来呈现分歧：

```
你：我们是一家快速增长的创业公司，正面临压力要强攻一个监管环境
非常敌对的城市，律师说风险很高。该强推还是该等？

Claude：同时载入两个视角，会呈现出真实的分歧，而不是给你一个
折中的答案：

- Kalanick 的框架指向强推：他公开表述的市场逻辑明确地把监管
  敌意当作一个正面信号——挺过越难的仗，护城河就越深。
- Hastings 的框架指向相反方向：他的原则是建立充分的上下文，
  让良好的判断（包括律师的判断）发挥作用，而不是用个人的确信
  去压过它——他自己已记录的失败案例（Qwikster）恰恰就是他的
  确信压过了正在提出担忧的人。

这两者之所以冲突，是因为它们对同一个信号——"来自权威的阻力"——
给出了相反方向的权重。这种张力本身就是有用的信息，不是要替你
解决的问题。
```

三份基于真实研究生成的完整档案——一个材料极其丰富的人物、一个材料确实单薄的人物、一个有真实公开争议的人物——都在 [`profiles/`](profiles/) 目录里，你可以在自己动手运行之前先看看实际产出是什么样子。

## 安装

这是一个 Claude Skill——一份纯指令文件（`SKILL.md`），没有代码依赖。

**Claude Code（命令行）：**
```
mkdir -p ~/.claude/skills/borrowed-brain
cp SKILL.md ~/.claude/skills/borrowed-brain/
```
（如果只想在某个特定项目里使用，把 `~/.claude/skills/` 换成该项目下的 `.claude/skills/borrowed-brain/` 即可。）

**Claude.ai：** 打开 Settings → Capabilities → Skills，直接上传 `SKILL.md`。

无论哪种方式，重启会话后提到某个人的名字即可触发——例如"帮我建一份 <某人> 的思维档案"。不需要 API key，不需要额外配置——Claude 会实时利用你所在环境里已有的联网搜索/抓取工具来完成研究和写作。如果环境里没有这类工具，skill 会明确告知你，而不是悄悄靠记忆编造。

skill 也会跟随你对话使用的语言输出——用简体中文和它对话，生成的档案和 Apply 模式的回应也会是简体中文。

## 护栏（内置，不可选）

- 不编造引用；直接引用限 15 词以内、每个来源最多引一句，其余全部转述
- 档案里的每一条声称都必须能追溯到一个真实找到的来源
- 措辞统一是"根据公开材料，其做法似乎是……"——不会写成"其相信……"这种既定事实的口吻
- 拒绝为私人（非公众人物）建档——只处理有实质公开记录的人
- 争议人物：只描述其已记录的推理风格和立场，不推测未公开的私人动机

## 这个 skill 不是什么

- 不是用来生成假语录或替人编话的工具
- 不能替代阅读这个人的真实著作——如果你想知道他们真正的观点，去读他们的原文；这个工具是用来找一个你可能忽略的**角度**
- 不是判决生成器——它被设计成每次应用都要以"这个视角覆盖不到什么"收尾，这是刻意的
